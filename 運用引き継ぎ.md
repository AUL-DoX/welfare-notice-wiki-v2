# 福祉通知ウィキ 運用引き継ぎ

## 1. 概要

- アプリ名: 福祉通知ウィキ
- ローカル作業フォルダ: `C:\welfare-notice-wiki`
- GitHub: `https://github.com/AUL-DoX/welfare-notice-wiki`
- Vercel標準URL: `https://welfare-notice-wiki.vercel.app`
- 本番URL: `https://wn-wiki.aul-dox.jp`

このアプリは、障害福祉・介護関係の通知文書を一覧化し、検索・詳細表示できる Next.js アプリです。

運用方針:

- 一般利用者は閲覧専用（検索・元ファイルを開く）
- 管理者のみ資料追加、カテゴリ設定、キーワード設定を行う
- 本番公開は Vercel
- データ反映は GitHub への push で行う（Obsidian Git が自動化）

---

## 2. フォルダ構成

```
welfare-notice-wiki/
├── source-docs/          ← 元資料（PDF等）を置く場所
│   └── meta/             ← Obsidian保管庫（カテゴリ・キーワード管理）

└── .env.local            ← 管理者パスワード（gitに含まれない）
```

## 3. 日常運用：資料を追加する

### 手順

1. **ファイルをリネームしてから** `source-docs` に置く
   - 政府サイトからダウンロードしたファイルは crypticな名前のことが多い
   - **必ず人が読める名前に変更してから置く**（例: `r8hokkaido_syougai.pdf` → `令和8年度_北海道障害福祉従事者処遇改善_実施要綱.pdf`）
   - ファイル名がそのままObsidianのノート名になる

2. **Obsidianで `meta` 保管庫を開き、同名の `.md` ファイルを新規作成する**
   - ファイル名: 元ファイルの拡張子を `.md` に変えたもの
   - 例: `令和8年度_北海道障害福祉従事者処遇改善_実施要綱.md`

3. **frontmatter（プロパティ）にカテゴリとキーワードを設定する**

   ```yaml
   ---
   category: 障がい福祉
   keywords:
     - 処遇改善
     - 賃上げ支援
     - 北海道
   ---
   ```

4. **`data/document-metadata.json` にアップロード日を追記する**
   - Vercel本番ではファイル書き込みができないため、手動でJSONに追記してgit pushする
   - slugの生成ルールは後述

5. **Obsidian Git が自動で push する（最大5分）**
   - Vercel が自動デプロイ（1〜2分）
   - 本番URLで確認

---

## 4. カテゴリ設定

### 設定場所

`source-docs/meta/資料名.md` の frontmatter で設定します。

```yaml
---
category: 障がい福祉
---
```

### 使用できる値（日本語・英語どちらでも可）

| 日本語 | 英語 | 意味 |
|------|------|------|
| `障がい福祉` または `障害福祉` | `disability` | 障害福祉サービス関連 |
| `介護` | `care` | 介護保険関連 |
| `共通` | `common` | 両方に共通 |
| `未分類` | `unclassified` | 未設定（デフォルト） |

## 5. 関連キーワード設定

### 設定場所

`source-docs/meta/資料名.md` の frontmatter で設定します。

```yaml
---
keywords:
  - 処遇改善
  - 賃上げ支援
  - 障害福祉サービス
---
```

### ルール

- `#` は不要（Obsidianのプロパティとして入力するだけ）
- 同じキーワードが別資料に重複しても問題なし
- 1資料あたり最大10個が表示される
- **設定したキーワードのみ表示される**（自動抽出は検索にのみ使用、表示はしない）
- キーワードは検索にも使われるため、利用者が検索しそうな言葉を入れるとよい

## 6. アップロード日の管理

Vercel本番ではファイル書き込みができないため、アップロード日は `data/document-metadata.json` に手動で追記してgit pushする必要があります。

### slugの生成ルール

slugはファイル名（拡張子なし）から以下のように生成されます：

1. Unicode正規化（NFKC）・小文字変換
2. 全角文字 → 半角（`Ａ`→`a`、`（`→`(`など）
3. 文字・数字以外はすべて `-` に変換
4. 連続する `-` はまとめる
5. 先頭・末尾の `-` を削除

**例:**
```
令和8年度_北海道障害福祉従事者.pdf
  → 令和8年度-北海道障害福祉従事者

介護職員等処遇改善加算に関するＱ＆Ａ（第１版） (2).pdf
  → 介護職員等処遇改善加算に関するq-a-第1版-2
```

### document-metadata.json の追記方法

```json
{
  "既存のslug": { "uploadedAt": "2026-04-15T00:00:00.000Z" },
  "新しいslug":  { "uploadedAt": "2026-04-16T00:00:00.000Z" }
}
```

追記後に git push すれば本番に反映されます。

## 7. Obsidian Git の設定

### 保管庫

`C:\welfare-notice-wiki\source-docs\meta` を Obsidian の保管庫として使用。

### プラグイン設定

- プラグイン名: **Git**（作者: Vincent, Denis Olehov）
- Auto commit-and-sync interval: `5`（分）
- → 編集後5分以内に自動でGitHubへpushされる

### 動作の流れ

```
Obsidianでfrontmatterを編集・保存
  ↓（最大5分）
Obsidian Gitが自動commit & push
  ↓
GitHubに「Vault Backup: 日時」のコミットが入る
  ↓（1〜2分）
Vercelが自動デプロイ
  ↓
本番サイトに反映
```

## 8. metaファイルの命名ルール

**metaファイル名 ＝ 元ファイル名の拡張子を `.md` に変えたもの**


⚠️ ファイル名が一致しないと frontmatter が読み込まれません。

## 10. Vercel 運用

### デプロイ確認

GitHub の各コミットの右に ✓（緑）または ✗（赤）のマークが出ます。

- ✓ 緑: デプロイ成功
- ✗ 赤: エラー（Vercel ダッシュボードで詳細確認）

Vercel ダッシュボード: https://vercel.com （要ログイン）

### Vercelの制限

本番では実行中にローカルファイルへの書き込みができません。

そのため以下は **本番では直接保存できません**:

- `document-metadata.json` への書き込み
- `document-categories.json` / `document-keywords.json` への書き込み（現在は使用していない）

## 11. PDF表示について

政府・自治体のPDFは内部構造が複雑なため、全文表示が崩れることがあります。

- 文字が1文字ずつ縦に並ぶ
- 文字が重複して表示される

これは元PDFの作成方法の問題であり、コードでの完全修正は困難です。

**対応方針:** 全文表示が崩れていても検索・キーワード・カテゴリは正常に機能します。利用者には「元ファイルを開く」ボタンで元のPDFを参照してもらいます（全文セクションに注意書きを表示済み）。

## 12. トラブル時の切り分け

### 本番だけおかしい

- GitHub のコミットに ✓ がついているか確認
- Vercel ダッシュボードで最新デプロイが `Ready` か確認

### 独自ドメインだけおかしい

- お名前.com 側の DNS を確認
- `wn-wiki` の CNAME 設定を確認

### Obsidian で編集したのに反映されない

1. Obsidian Git パネル（左サイドバーのブランチアイコン）を確認
2. `Ctrl+P` → `Git: Commit-and-sync` で手動push
3. GitHub のコミット履歴に「Vault Backup」が入っているか確認
4. **metaファイル名と元ファイル名が一致しているか確認**（最もよくある原因）

### アップロード日が 1970/01/01 になっている

- `data/document-metadata.json` にそのファイルのslugが登録されていない
- slugを生成して追記し、git push する
